# HardSecNet Development Plan

## 1. Development Strategy

Build the remaining product in vertical slices.

Do not build all backend first, then all UI, then all agent. That creates too much dead code.

Each epic should produce something visible and testable.

## 2. Recommended Epic Order

1. Benchmark Native Mode
2. Benchmark Browser And Control Review
3. Script Registry And Validation Workflow
4. Dashboard Operator Workflows
5. Agent Loop And Offline Resilience
6. Report And Artifact Access
7. Validated Hardening Modules
8. Production Packaging And Release Hardening

## 3. Engineering Rules

- Do not execute candidate scripts as hardening.
- Add tests for every new state transition.
- Keep local desktop mode working.
- Keep dashboard build passing.
- Keep FastAPI tests passing.
- Store benchmark provenance with every control.
- Keep docs updated when a workflow changes.

## 4. Milestone 1: Benchmark Native Mode

Goal:

- app loads repo-exported CIS bundles automatically

Deliverables:

- bundle discovery
- JSON schema validation
- local repository upsert
- duplicate detection
- tests

Verification:

- fresh runtime DB contains Windows and Ubuntu documents after bootstrap
- no PDF required

## 5. Milestone 2: Benchmark Browser

Goal:

- user can inspect controls from the desktop app

Deliverables:

- benchmark page/table
- search/filter
- control detail panel
- script preview
- source/provenance display

Verification:

- UI smoke test includes benchmark browser
- tests confirm controls load from exports

## 6. Milestone 3: Script Registry

Goal:

- generated scripts have explicit lifecycle state

Deliverables:

- `ScriptRecord`
- persistence table
- state transitions
- review metadata
- execution gate

Verification:

- hardening job rejects non-validated scripts
- audit-only jobs can reference candidates safely

## 7. Milestone 4: Dashboard Operator Workflows

Goal:

- dashboard supports real operator actions

Deliverables:

- login screen
- device detail
- job creation
- approvals
- report detail

Verification:

- dashboard build passes
- API tests cover job creation/approval
- basic browser e2e added if feasible

## 8. Milestone 5: Agent Loop

Goal:

- agent runs continuously

Deliverables:

- loop mode
- polling interval
- config file
- offline queue
- retry/backoff
- graceful shutdown

Verification:

- fake backend test covers retry
- fake adapter test covers repeated polling

## 9. Milestone 6: Reports And Artifacts

Goal:

- reports are accessible from dashboard and API

Deliverables:

- report detail endpoint
- download endpoint
- artifact index

- dashboard report viewer

Verification:

- report generated by job can be downloaded

## 10. Milestone 7: Validated Modules

Goal:
